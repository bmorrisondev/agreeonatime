# Build in public (DEV-400)

Suggested cadence for Agree on a Time v1.0 launch:

| Day | Deliverable |
|-----|----------------|
| 1 | Kickoff post — spec link, deadline, stack (Expo + Convex) |
| 2–6 | Short devlogs: auth, events, web vote, pick-time |
| 7 | 60–90s screen recording: create → share → vote → decide |
| 10 | EAS / TestFlight + live marketing URLs |
| 12 | Submitted to App Review |
| 14+ | Launch post + App Store link |

Pick 1–2 channels (e.g. Bluesky, X, blog). Keep posts factual.
